
# 👋 Welcome to MauroGioberti.com 🌐

Hey there! Welcome to the repo for **maurogioberti.com** — my personal space on the web. Here, I’m setting up a blog, my resume, some background about myself, and LinkedIn recommendations in a project driven by Clean Architecture principles (even on the frontend!). This site is a labor of love, and if you’re here, you might be curious about coding, personal projects, or clean architecture. Great to have you!

## 💡 Project Overview

This is a **Next.js** project with a twist: I’m using a layered architecture for the frontend, inspired by Clean Code principles. The idea is to keep the logic well-separated, easy to maintain, and clean enough that anyone who drops by can easily get involved. The backend? Well, I’m emulating it with mocked JSON responses server-side, so you can get a feel for the data flow without needing a full server setup.

This repository is open to anyone curious, so feel free to explore, fork, or contribute if you’re inspired!

## 🚀 Getting Started

Ready to dive in? Here’s how you can get the development server up and running:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
```

Once it’s running, open [http://localhost:3000](http://localhost:3000) to check out the site locally. You can start editing by modifying `src/app/page.tsx` — it updates automatically as you make changes!

### Tech Stack & Tools

- **Framework**: Next.js (TypeScript, of course)
- **Styling**: Tailwind CSS
- **Architecture**: Clean Architecture applied to frontend
- **Backend**: Mocked JSON responses to simulate API calls
- **Deployment**: Vercel for easy, fast deployment

## ✨ Key Features

- **Clean Code Architecture**: Every layer has its role, keeping components and logic separate for clarity and maintainability.
- **Public Repository**: Since this repo is public, it’s a place to showcase this architecture approach and maybe even inspire others to give it a shot.
- **Mocked Backend**: JSON mocks on the server-side to simulate backend responses, making it easy to test functionality without needing a backend.
